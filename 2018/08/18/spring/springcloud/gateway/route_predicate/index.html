Spring Cloud Gateway 路由断言

发表于 2018-08-18 | 分类于 Spring ， Spring Cloud ， Spring Cloud Gateway | 阅读数
字数统计: 838 | 阅读时长 ≈ 3

简介

Spring Cloud Gateway 的路由匹配是基于 Spring WebFlux 的 HandlerMapping 组件构建。Spring Cloud Gateway 内置很多 Route Predicate Factory，这些 Predicate Factory 都是基于 Http 请求中的不同属性构建。并且，可以通过 and 对多个工厂进行逻辑组合。

Spring Cloud Gateway 是由多个路由断言工厂组成的，当请求进入 Spring Cloud Gateway 时，系统中的路由断路器工厂根据配置的路由规则，对请求进行断言匹配。匹配成功则进入下一步处理，否则直接返回错误信息。

After Route Predicate

以一个 UTC 时间作为参数，当当前时间在设置的 UTC 时间之后时，进行匹配。

匹配在 2017-01-20T17:42:47.789-07:00[America/Denver] 之后发生的请求。

Before Route Predicate

以一个 UTC 时间作为参数，当当前时间在设置的 UTC 时间之前时，进行匹配。

匹配 2017-01-20T17:42:47.789-07:00[America/Denver] 之前发生的请求。

Between Route Predicate

以两个 UTC 时间作为参数，当当前时间在设置的时间段之间时，进行匹配。

匹配 2017-01-20T17:42:47.789-07:00[America/Denver] 和 2017-01-21T17:42:47.789-07:00[America/Denver] 之间发生的请求。

Cookie Route Predicate

需要两个参数，cookie name 和要匹配的表达式。当请求中存在 cookie name 并且对应的 value 匹配表达式时进行匹配。

匹配 cookie 中存在 chocolate，并且对应的值匹配 ch.p 的请求。

Header Route Predicate

需要两个参数，header name 和要匹配的表达式。当请求中存在 header name 并且对应的 value 匹配表达式时进行匹配。

匹配 header 中存在 X-Request-Id，并且对应的值匹配 \d+ 的请求。

Host Route Predicate

需要一个 Host 正则列表，当请求的 Host 与列表中的 Host 正则匹配时，进行匹配。

匹配请求头中 Host 值与 .somehost.org, .anotherhost.org 正则匹配的请求。

Method Route Predicate

根据配置的 method 方式进行断言处理。

匹配请求方法为 Get 的请求。

Path Route Predicate Factory

根据请求路径进行断言处理。

匹配请求路径与 /foo/{segment},/bar/{segment} 匹配的请求。

Query Route Predicate Factory

更加查询参数进行断言处理。

匹配请求参数含有baz的请求。

匹配请求参数含有foo，其值匹配 ba. 的请求。

RemoteAddr Route Predicate Factory

配置一个 IPv4 或 IPv6 的网段字符串，当请求 IP 地址在网段之内，进行匹配。

匹配 IP 在 192.168.1.1/24 间的请求。
