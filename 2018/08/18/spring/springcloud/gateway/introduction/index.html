Spring Cloud Gateway 简介

发表于 2018-08-18 | 分类于 Spring ， Spring Cloud ， Spring Cloud Gateway | 阅读数
字数统计: 947 | 阅读时长 ≈ 4

网关提供了 API 管理服务，极大丰富了 API 管理功能，降低了成本和安全风险。其中主要包括协议适配、协议转发、安全控制、防刷、限流、监控等功能。

Spring Cloud Gateway 是构建于 Spring 5、Spring Boot 2 和 Reactor 等框架开发的网关。旨在为微服务架构提供简单、有效、统一的 API 网关管理。最为 Spring Cloud 的第二代网关，目标是替换 Netflix Zuul 组件。其不仅提供路由管理功能，还以 Filter 的方式提供横切关注点功能，如：安全、监控/埋点、限流等。

启用

最简单的启用 Gateway 的方式是使用 gateway 的 starter，如果不想启用，可以通过配置禁用 gateway 功能。

添加 starter-gateway

禁用 gateway 功能：

Spring Cloud Gateway 依赖于 Spring Boot 和 Spring Webflux 所提供的 Netty 运行环境，它无法在一个 Servlet 容器中运行，也无法以 War 的方式运行。

核心组件

Gateway 核心组件包括：

  1. Route 是 Gateway 的最核心组件，它由 ID、Url、一组 Predicate 工厂和 一组 Filter 集合构成。当 Predicate 为真时，则说明该请求与路由匹配。
  2. Predicate 是 Java 8 中的函数接口。Spring Cloud Gateway 中的断言输入类型为 ServerWebExchange。它允许开发人员匹配来自 Http 的任何信息，如 Header 或 Parameter。
  3. Filter 是使用特定工厂创建的 GatewayFilter 实例，可以在下一个 Filter 执行前和执行后，对请求和响应进行修改。

核心流程

核心流程如下：

  1. 客户端向 Spring Cloud Gateway 发送请求；
  2. Gateway Handler Mapping 判断该请求与路由是否匹配；
  3. 如果与路由匹配，则将请求发送至 Gateway Web Handler；
  4. Gateway Web Handler 将请求转发给过滤器链；
  5. 挨个执行过滤器的 pre 逻辑；
  6. 所有的 pre 逻辑执行完成后，执行代理请求；
  7. 代理请求完成后，挨个执行过滤器的 post 逻辑；
  8. 最后将结果返回给 Client

案例

作为网关来说，最主要的功能就是协议适配和协议转发，协议转发就是指基于路由信息转发。

Spring Cloud Gateway 提供了两种方式进行路由配置。一种是基于 Java 流式 API进行定义；一种是基于配置文件的配置。
